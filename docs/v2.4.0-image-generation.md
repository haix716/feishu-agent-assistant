# v2.4.0 图片生成 — 设计文档

## 目标

用户上传商品图/服装图，智能体自动生成：
1. **穿戴效果图** — 真人模特穿着上传的服装
2. **商品详情图** — 电商风格的商品主图/封面图

## 技术架构

### 整体流程

```
用户发图
  ↓
图片分析（MiMo 多模态）
  ↓
┌─ 识别内容类型：服装 / 商品 / 其他
├─ 提取关键信息：颜色、材质、品类、风格
└─ 生成提示词（中→英）
  ↓
路由到对应生成流程
  ├─ 穿戴 → Replicate Try-On API
  ├─ 商品图 → 即梦 API（img2img）
  └─ 封面图 → 即梦 API（img2img + 构图）
  ↓
返回生成结果
  ├─ 保存到本地
  ├─ 上传飞书云盘（可选）
  └─ 发送图片给用户
```

### 核心接口

```typescript
// providers/provider.ts
interface ImageProvider {
  name: string;
  generateImage(params: GenerateParams): Promise<GenerateResult>;
}

interface GenerateParams {
  referenceImage: Buffer;      // 参考图（用户上传的原图）
  prompt: string;              // 英文提示词
  negativePrompt?: string;     // 反向提示词
  mode: 'tryon' | 'product' | 'cover';
  options?: {
    modelType?: string;        // 模特类型（穿戴用）
    aspectRatio?: string;      // 宽高比（封面用）
    style?: string;            // 风格（场景图用）
    numImages?: number;        // 生成数量
  };
}

interface GenerateResult {
  images: Buffer[];            // 生成的图片
  revisedPrompt?: string;      // API 修改后的提示词
  cost?: number;               // 本次花费
}
```

### Provider 设计

**Replicate Provider**（穿戴 + 通用）：
- Try-On 模型：`tencentarc/photomaker` 或 `idm-vton/idm-vton`
- 通用 img2img：`black-forest-labs/flux-dev`
- API 格式：REST，轮询结果

**即梦 Provider**（商品图 + 封面）：
- 通过火山引擎 API 调用
- img2img 接口
- 支持中文提示词

### 图片分析（analyzer.ts）

用现有的 MiMo 多模态模型分析图片：

```typescript
interface ImageAnalysis {
  contentType: 'clothing' | 'product' | 'accessory' | 'other';
  category: string;        // 具体品类：T恤、保温杯、项链...
  attributes: {
    color: string;         // 主要颜色
    material: string;      // 材质
    style: string;         // 风格：简约、复古、可爱...
    brand?: string;        // 品牌（如果有）
  };
  suggestedPrompt: string; // 自动生成的英文提示词
  suggestedMode: 'tryon' | 'product' | 'cover';
}
```

### 提示词模板（prompt-builder.ts）

**穿戴效果图**：
```
A {ethnicity} {gender} model wearing a {color} {material} {clothing_type},
{style} style, {pose}, natural lighting, professional fashion photography,
full body shot, high quality, 4K
```

**商品主图（白底）**：
```
Product photography of a {color} {product_type}, clean white background,
professional studio lighting, commercial product photo, high resolution,
centered composition
```

**商品场景图**：
```
A {product_type} placed in a {scene} setting, {style} aesthetic,
natural lighting, lifestyle photography, shallow depth of field,
high quality, 4K
```

**小红书封面**：
```
Aesthetic product photography of {product_type}, {style} style,
soft lighting, Instagram-worthy composition, trendy, visually appealing,
high quality, 3:4 aspect ratio
```

## 交互流程（handler.ts 更新）

### 图片消息处理流程（更新）

```
用户发图片
  ↓
handleImageMessage（现有）
  ├─ 下载图片
  ├─ 分析图片内容（analyzeImage）
  └─ 保存到本地（现有）
  ↓
新增：图片意图识别
  ├─ 问用户要生成什么（默认）
  ├─ 如果用户发图时带了文字指令 → 直接路由
  └─ 等待用户回复 → 解析意图 → 调用生成流程
```

### 文本消息路由（新增）

在 `handleMessage` 中新增图片生成关键词检测：

```typescript
// 关键词触发（快速路径）
const imageGenKeywords = /^(穿戴|试穿|生成.*图|做.*封面|商品图|主图)/;
if (imageGenKeywords.test(query) && pendingImages.has(userId)) {
  await handleImageGeneration(channel, msg, pendingImages.get(userId)!);
  return;
}
```

## 风险与应对

| 风险 | 影响 | 应对 |
|------|------|------|
| 即梦 API 不支持 img2img | 无法做商品图 | 用 Replicate 替代 |
| Replicate 免费额度用完 | 无法测试 | 用硅基流动替代 |
| 生成质量不稳定 | 用户体验差 | 多生成几张让用户选 |
| API 响应慢（>30s） | 用户等待焦虑 | 先发"生成中"提示，轮询更新进度 |
| MiMo 图片分析不准 | 路由错误 | 兜底：让用户手动选择 |
